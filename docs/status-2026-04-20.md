# Elide Status — 2026-04-20

Progress since [2026-04-09](status-2026-04-09.md).

---

## Major changes

### Thin delta compression — end-to-end

The full delta stack landed in phases.  Segment format gained a thin
`EntryKind::Delta` that stores a source-hash reference plus a compressed
patch; the shared body section now carries a dedicated delta body region.
File-aware import walks the ext4 extent tree to emit per-fragment DATA
entries, which the filemap module indexes at snapshot time so the
coordinator can offer a dictionary source.  A zstd-dictionary delta repack
runs post-snapshot in a worker thread, compressing cold bodies against
near-neighbour sources.  Read path decompresses thin Delta entries
transparently; pull-host demand-fetch resolves the delta body section over
range-GET.  Deltas are preserved through GC compaction.

### Thin DedupRef upload + dead-entry hole-punching

DedupRef is now a format-level thin entry that carries only a source-hash
pointer, not a materialised body copy.  Segments can be rewritten in place
to punch holes for LBA-dead but hash-unreferenced entries (redact_segment).
GC compaction preserves DedupRef entries rather than materialising them,
holding the earliest-source preference stable.  Signed provenance +
volume.provenance consolidated the scattered lineage files; provenance
signature verification is tested against tampering.

### Self-describing GC handoff protocol

The `.pending`/`.applied` suffix scheme is replaced with a self-describing
protocol.  Segment format v5 records input ULIDs inside the compacted
segment header; the coordinator writes `.staged` handoffs and the volume
derives what to apply from the segment's own inputs.  The legacy
`elide-core::gc` module is deleted.  The TLA+ model of the handoff was
rewritten to match the new protocol and crash-bounded to prove
`EventuallyDone`.

### Actor worker-thread offload

The single flusher thread was generalised into a worker with enum job
dispatch.  Long-running or blocking work moved off the actor loop:
`promote_segment`, `sweep_pending`, `repack`, `delta_repack_post_snapshot`,
`sign_snapshot_manifest`, GC handoff re-sign, and WAL fsync all run on the
worker.  The actor stays responsive to NBD I/O while these run.  A
segment-index LRU is shared between actor and worker.

### GC partial-death compaction

Previously GC skipped any segment with partial-LBA-death entries.  Now
Data/Inline, DedupRef, and Delta entries all support partial-death
compaction: surviving sub-runs are split out and bodies expanded via a
cross-segment resolution helper.  Zero entries split into surviving
sub-runs on liveness.  Dead-LBA canonical entries are demoted to
`FLAG_CANONICAL_ONLY` instead of being dropped, preserving hash liveness
for dedup references.

### Extent reclamation primitive

First-cut extent reclamation: an alias-merge primitive, a candidate scanner
over the LBA map + extent index, and `elide volume reclaim` CLI + IPC.
The design doc was rewritten around two primitives (alias-merge and
defragmentation).  Reclaim runs as a SimOp in proptest.

### Fork from remote

`volume fork --from <vol_ulid>/<snap_ulid>` now auto-pulls ancestor
snapshots from the remote object store when they're not local.  Pulled
ancestors land in a `readonly/` tree to make their lifecycle explicit.
Demand-fetch routes through the ancestor's index dir.

### Coordinator-driven snapshot + provenance upload

Snapshot publishing moved under the coordinator: signed snapshot manifest
written during import, snapshot markers and filemaps uploaded to S3,
prefetch on pull.  Local snapshot is used for readonly volumes.
First-snapshot pinning invariant asserted in `snapshot()`.

### Write-path no-op skip

Same-LBA same-content writes now short-circuit before the WAL.  Two-tier
check: hash compare first, then opportunistic byte compare for the common
small-write case.  Covered by a `SameContentWrite` SimOp.

### CLI and lifecycle

- `elide volume stop` / `volume start` replace the earlier `up`/`down`
  naming; `volume up`/`down` removed.
- NBD endpoint conflict detection between volumes; `volume stop` refuses
  while an NBD client is connected.
- `volume create --from` for convenience forking; bare ULID supported.
- `volume info` gained per-entry-kind breakdown, extent sources, cached
  ratio, delta entry counts.

### Sweep bin-packing

Sweep packs outputs toward 32 MiB live bytes with a filler tier, replacing
the earlier one-segment-per-input shape.  Import aligns segment size with
the runtime target.

### elide-fetch: drop tokio

`elide-fetch` switched to synchronous `rust-s3` and dropped its tokio
dependency, bringing the fetch crate in line with the sync volume binary.

### Idle-tick quieting

WAL creation deferred to first write to stop idle GC tick churn; GC skips
the checkpoint on idle volumes; routine dead-entry logs demoted to debug.

## Bug fixes

Notable correctness fixes (most via proptest or the TLA+ model):

- **GC body hash verification before rewrite** — a stale-liveness bug was
  silently producing zero-filled compacted extents; compaction now
  hash-checks each body before writing it.
- **Refcount delta source hashes in lbamap** — GC was cancelling in a loop
  when a delta's source hash had no live LBA but was still referenced.
- **Coordinator liveness walks ancestor chain** — rebuild was missing
  ancestor-provided hashes, causing false dead-entry classification.
- **WAL replay into coordinator LBA map** — prevented stale-liveness loop
  after crash.
- **GC rebuild skips consumed input idx files** — kept the coordinator and
  volume views consistent across handoff restarts.
- **Supersede filter inside discover_fork_segments** — rebuild was keeping
  superseded segments alive in fork-aware passes.
- **apply_gc_handoffs serialisation** — dropped reply senders on
  concurrent invocation; now serialised.
- **Bare `gc/<U>` processed at own ULID on rebuild** — previously sorted as
  lowest-priority and lost its place in the ancestry.
- **Volume tmp-sweep vs coordinator `staged.tmp`** — scratch-file races
  between the two writers resolved with a single-writer invariant.
- **fetch_extent durability** — `.body` now written before `.present` so
  a crash mid-fetch can't leave a present-marker with no body.
- **promote writes `.body` last** — commit point for the whole promote.
- **Preserve sparseness in promote_to_cache**; reduced allocations in
  materialise_segment.
- **Deduplicate filemap entries** for files with identical content during
  import.
- **evict cached fd on rename-replaced pending segment** — otherwise stale
  fds returned pre-rename data after sweep.
- **invalidate extent index for hash-dead punched entries** — redact
  didn't remove stale index pointers.
- **dispatch Delta entries in BlockReader::read_block** — read path was
  falling through to a DATA-only branch.

## Proptest and model improvements

- `SameContentWrite` SimOp covers the no-op write skip.
- `reclaim` unified into the SimOp strategy.
- Multi-LBA overwrite SimOp + deferred count in `GcStats`.
- TLA+ model of GC handoff rewritten for the self-describing protocol;
  crash count bounded so `EventuallyDone` holds.
- Invalidate stashed `GcCheckpoint` on a full GC pass (regression guard).
- Promote-segment mid-apply crash-recovery regression tests.
- Offload test coverage for sign-snapshot + delta-repack, modelling the
  actor↔worker boundary.

## Remaining work (updated)

Item 1 from the previous status (delta compression) has landed end-to-end,
including the coordinator upload path and pull-host demand-fetch.  Extent
reclamation is now wired into the per-tick drain loop (one candidate per
tick, scanner-gated); only threshold tuning remains.

Remaining priorities:

1. **Boot hints + S3 repacking** — still depends on access-pattern
   telemetry; repacking heuristic sketched in the delta doc as a hint.
2. **ublk integration** — production Linux block device target.
3. **Snapshot indexes** — acceleration structure for cold `Volume::open()`.
4. **Reclamation thresholds** — `ReclaimThresholds` defaults are
   placeholders. Picking a real "worth firing" bar (and a per-tick cap
   above 1) needs empirical work on an aged volume — same telemetry
   dependency as item 1.
5. **Inline extents** — threshold still 256 bytes; low priority until
   fetch-path telemetry says otherwise.
