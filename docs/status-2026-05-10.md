## Major changes

### Peer-fetch â€” cross-host body routing

The biggest landing in this window.  A claiming host can pull live extent
bodies directly from the previous owner instead of always rehydrating
through S3.  Wire surface, auth, and routing all landed:

- `PeerRangeFetcher` decorator wraps the demand-fetch path; volume
  daemon serves a volume-signed `.body` route with partial-coverage
  `206` responses and a per-bearer name-record sub-cache.
- Manifest-only warm-start: ancestor `.idx` fetch set is driven from
  the snapshot manifest; `.filemap` dropped from the wire surface and
  no longer uploaded to S3 (filemaps are strictly local).
- Demand-fetches coalesce concurrent overlapping batches; chain walk
  removed from `RemoteFetcher` â€” fetches target the segment's owner
  directly.
- Body prefetch on claim (v1.2): `.prefetch` hints persisted across
  restart; warm bodies on volume start; parallel-batch single-fsync
  warming path.  Peer-fetch disables itself once hint-driven warmup
  completes.
- Cross-host body-route integration tests + per-volume body-byte
  counters.  Skeleton routes relaxed via `RouteAuthMode` with a
  rate-limit hook.

### Volume event log

Per-name event log under `names/<name>/events/`, replacing ad-hoc
state inference:

- `VolumeEvent` (renamed from NameEvent) emitted from lifecycle call
  sites: `Claimed`, `ForkedFrom`, release, etc.
- Hostname stamped on every event from cached identity.
- Claim-discovery walks the log to find the previous owner for
  peer-fetch handoff; per-event body fetches parallelised in
  `list_events`.

### NBD transport removed

NBD is gone.  ublk is the sole Linux block transport.  CLI surfaces,
volume.toml sections, supervisor wiring, and tests for NBD all
deleted.

### GC / compaction â€” unified pass

The three compaction primitives collapsed into one:

- **sweep + repack unified** into a single bin-packing rewrite pass;
  redact folded into repack as a special case (rewrites under fresh
  ULID rather than in-place hole-punch).
- **N-bucket FFD bin-pack with cache-residency filter** â€” selector
  considers cache state so warm segments don't get demand-fetched on
  rewrite.  All pending segments are bin-packed in one go; drain
  visibility preserved.
- `gc_plan`/`gc_apply` renamed to `rewrite_plan`/`rewrite_apply`;
  per-entry liveness classifier extracted; sweep / repack /
  delta_repack / redact migrated to a uniform
  classifier+plan+materialise shape with fresh ULID per output.
- `apply_gc_handoffs` ancestor walk and partial-death classification
  fixed; `Canonical` emitted in the test helper to match production.

### LBA-map claimant tracking + runtime invariants

Two coupled correctness pieces:

- LBA-map entries now carry the claimant ULID; sweep/repack/redact
  apply uses `insert_if_newer` with a consumed-input override for
  parallel claimants.  Fixes a sweep+repack data-loss bug
  (`finding-sweep-flush-claimant-bug.md`) where the claimant guard
  rejected legitimate winning rewrites.
- Runtime invariants gated behind `--features proptest`:
  `assert_lbamap_hashes_resolvable`, `assert_extent_index_consistent`,
  `assert_pending_above_committed`.  The rebuild-from-disk path is
  the correctness oracle; incremental updates must structurally match
  it.

A separate cargo-build stale-read corruption was reproduced and
documented in `finding-cargo-build-stale-read.md`.

### Delta materialisation cache (`.dmat`)

New per-segment sidecar that caches materialised delta bodies between
reads.  `dmat` module in elide-core, sidecar wired into the delta
read path with telemetry counters, torn-write recovery proptest, and
an `inspect-dmat` diagnostic command.

### Performance pass

Coordinated pass on the read/write hot paths:

- **Data-structure swaps.**  `BTreeMap` â†’ `imbl::OrdMap` for the LBA
  map (`Arc::make_mut` is now O(log N) not O(N)).  Inner `HashMap` â†’
  `imbl` HAMT for the extent index.  Identity-style hasher for
  `blake3::Hash` keys (keys are already uniform).
- **Mutex layering.**  `Volume` wrapped in `Arc<Mutex<>>` shared with
  `VolumeClient`; mutex held weakly on the client side; ublk threads
  write directly through the volume mutex; `blake3` + `lz4` hoisted
  off the mutex.  `lock_volume()` centralised.
- **Read path.**  `pread` for hot-path segment + index reads; TLS
  scratch buffer with decompress-into-output; caller buffer plumbed
  through; in-memory presence bitset skips `find_segment_in_dirs` on
  the hot FD path.
- **Write path.**  Vectored WAL append with stack-allocated header;
  body bytes dropped from `pending_entries` (re-read via `pread` at
  promote); shannon-entropy gate dropped from `maybe_compress` (lz4
  is fast enough that the gate cost more than it saved); owned `Vec`
  handed down through `write_commit`.
- **Bench coverage.**  End-to-end `VolumeClient::write` throughput
  (single + multi threaded), extent-index `Arc::make_mut` clone cost,
  siphash vs blake3-id hasher lookup throughput.

### Coordinator process model

Lifecycle tightened around real production semantics:

- New verbs: `coord run` (foreground), `coord start` (renamed from
  `up`), `coord stop` (defaults to keeping volumes running; falls
  back to direct teardown if IPC is dead).
- Defensive shutdown: SIGTERM â†’ SIGKILL escalation when volume
  children hang, watchdog around `del_dev`, signalfd watcher.
- Unified `<data_dir>/elide.log` with per-process fds and tee to
  stderr; volume â†’ coord log relay socket so live tee survives
  rolling restarts.
- `data_dir` honoured from `coordinator.toml` in `run`/`start`/`stop`.
- `PidFileGuard` extracted; `setsid` dropped from import/fetch
  workers.

### Coordinator orchestration refactor

A multi-PR cleanup turning the coordinator from a soup of free
functions into typed orchestrators:

- `IpcContext` slimmed (17 â†’ 12 fields), then split into per-domain
  context types.
- `ClaimOrchestrator`, `ForkOrchestrator`, `GcCycleOrchestrator`
  extracted from inline call sites.
- Process-globals migration: peer-fetch handle, credential issuer,
  `[store]` config, socket path, rescan signal, multipart part size,
  bin paths â€” all converted from threaded args to process-globals.
  Knock-on simplification of inbound handlers and CLI plumbing.

### Credentials

- **Cold-start lazy fetcher.**  Credential issuance is deferred
  until first use; passthrough creds cached; S3 creds dropped after
  idle and re-acquired on demand.
- Coordinator scrubs `AWS_*` secrets when spawning volume
  subprocesses; credentials threaded explicitly rather than via env
  mutation.

### Force-release fencing + portable live volume

Foundations for cross-host volume migration without a coordinator:

- `design-force-release-fencing.md` reworked around snapshot-floor
  reuse â€” force-release recovers to "now" via signed segment replay.
- `design-portable-live-volume.md` + `portable-live-volume-plan.md`
  with Phase 0 foundations: snapshot manifest carries recovery
  metadata; `recovery::list_and_verify_segments` lands.
- `design-consistency-surface.md` catalogues the consistency surface
  and sketches a two-bucket split.

### Snapshot model

- **Manifest IS the snapshot record.**  Empty `snapshot/` marker
  dropped â€” the manifest is the authoritative snapshot record.
- Liveness predicate widened to catch body-bearing orphans.
- Fully-dead segments omitted from the manifest.
- `manifest.toml` removed; `oci_source` lives on `volume.provenance`.

### CLI

- `volume fetch <name>` warms a foreign volume locally without
  claiming it.
- `volume ls` removed; `volume evict`/`verify` hidden from the user
  CLI.
- `coord stop` defaults to keeping volumes running.
- ublk `--ublk-id` removed from the user CLI; auto-allocate by
  default; owner stamp verified on recover; `ublk.id` collapsed into
  `volume.toml`.
- `volume list` renders state without coordinator IPC when the
  coordinator is down.

### Other

- `design-deployment-modes.md` design proposal landed.
- `inspect-dmat` diagnostic command.

## Bug fixes

- **Sweep+repack data loss via LBA-map claimant guard** â€” the
  claimant invariant rejected legitimate winning rewrites; fixed by
  the consumed-input override (`finding-sweep-flush-claimant-bug.md`).
- **Cargo-build stale-read corruption** reproduced and root-caused
  (`finding-cargo-build-stale-read.md`).
- **Drain promotes pending segments in ULID-ascending order** â€” out
  of order promotion was producing classifier mismatches.
- **DATA-supersedes-Delta enforced in the live insert path** â€”
  previously only enforced at rebuild.
- **`apply_repack_result`** offload to a worker thread used a fresh
  ULID per output and per-input CAS apply.
- **Redact re-issues ULID + migrates extent index** to close a read
  race between rewrite and the next reader.
- **Classifier requires anchor agreement, not just hash**, for
  `matching_blocks`.
- **`DedupRef` liveness scans the entry's full range**, not just the
  head block.
- **Coord drain distinguishes S3-upload failure from volume-promote
  IPC race** â€” was producing false drain failures during coordinator
  rolling restart.
- **`fetch.pid` macaroon auth** binds fetch credentials to the
  worker PID.
- **Volume tears down its bound ublk device on remove**; ublk
  accepts whole-device discards larger than the per-tag buffer.
- **Claim emits `Claimed` (not `ForkedFrom`) on rebind**; treats
  released forks as not-running; publishes `volume.provenance`
  before claiming the name.
- **Fetch serialises `.present` read-modify-write** across disjoint
  leases; warm batches skip non-data entries.
- **`write_file_atomic`** uses a unique tmp suffix to avoid concurrent
  rename races.
- **`volume.toml` size written on the fetched dir** for `volume
  fetch`.
- **`payload_block_offset` preserved when splitting around blocked
  sub-runs** in the LBA map.

## Remaining work

prior status docs.  The transport surface is now ublk-only and the
S3-vs-peer body-fetch story is a real, tested path rather than a
sketch.

Open priorities, largely unchanged:

1. **Boot hints + S3 repacking** â€” still telemetry-gated.
2. **Snapshot indexes** â€” acceleration structure for cold
   `Volume::open()`.
3. **Reclamation thresholds** â€” the unified bin-pack still uses
   placeholder thresholds; needs empirical work on an aged volume.
4. **Portable live volume** â€” Phase 0 foundations are in; remaining
   phases (force-release, recovery replay) are the next big piece.
5. **ublk zero-copy** â€” still deferred until the rest of the
   transport stabilises.
